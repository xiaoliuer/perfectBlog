深度解析DomDiff算法（二）
发表于 2020-09-04| 更新于 2020-11-03|技术
阅读量:

了解了vdom和dom diff的前世今生，让我们来看看当今主流前端框架的diff算法策略又是什么，本篇我们先介绍react的dom diff算法

React算法时间复杂度

众所周知，react框架把diff算法复杂度由O（n的三次）降低到了O（n）

React对diff算法的改进

react的diff算法针对三个纬度产出了相应的优化策略：
1、web ui中跨层级的移动操作很少，可以忽略不计 （tree diff）
2、拥有相同类型的两个组件将会生成相似的树形结构，不同类型的组件生成不同的树形结构（component diff）
3、对于同一层级的一组子节点，通过唯一的key进行区分（ element diff）

一、继续深入三种策略

1、tree diff

两个策略：
1、对树进行分层比较，两棵树只会对比同一层级的节点
2、发现某一个节点不存在，直接删除改节点

那如果碰到跨层级操作怎么办？（一般不推荐这样做，因为需要全部销毁重建，很消耗性能，这种情况巧用css）

2、component diff

两个策略：
1、如果是同一类型组件，diff子节点
2、如果是不同类型组件，判定为dirty component，直接替换整个组件

3、element diff

对于节点提供五种操作
1、插入
2、移动
3、删除
4、文本内容变更
5、属性更改

其实三种diff策略好比从宏观到微观处理的过程

三、react的diff算法

基于上面的问题，react提出了基于key键值更优的diff算法


主要分四步：
1、遍历新节点，每个节点通过key来判断旧集合中是否存在相同节点，不存在的话直接新增；存在继续进行下一步
2、将当前节点在旧集合中的位置与lastIndex比较，当child._mountIndex < lastIndex时进行移动，移动到的位置即是nextIndex的位置（顺序优化策略）
3、进行步骤2后，更新lastIndex的值，此处有两步
lastIndex = Math.max（prevChild._mountIndex，lastIndex）
prevChild._mountIndex = nextIndex
（此处prevChild = B，B._mountIndex = 0）
 4、nextIndex++;

上面讲到的顺序优化策略就是：如果当前元素在旧集合中的位置比之前在新集合遍历过的所有节点在旧集合中的最大位置（lastIndex）还要大，那么说明它不会影响前面的节点位置，就不需要进行移动。（大家可以仔细品品这句话）所以就导致了，react diff中只有节点从前往后移动，没有往前移动的情况。

四、总结

1、react diff三条优化策略是基础
2、有key的element diff策略是核心
3、解决问题是关键
